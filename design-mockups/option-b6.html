AI-driven spec interview

WH-001 · Draft · Last edited 2 minutes ago

An AI-powered interview system that guides product owners, testers, and developers through developing comprehensive acceptance criteria for a feature. The interviewer has deep access to the target codebase to understand how changes fit together, identify edge cases, and find interactions with existing functionality.

Interview experience

  • AI conducts a structured conversation to elicit feature requirements, probing for edge cases, missing requirements, and contradictions
  • Interview happens within Workhorse's web UI as the primary interface
  • Uses a frontier model with full read access to the target codebase
  • Interview can be paused and resumed across sessions
  • Multiple people can participate in the same interview
  • AI signals when it believes the spec has reached sufficient detail

Codebase awareness

  • AI has read access to the target repository during the interview
  • Proactively identifies interactions with existing functionality when conversation implies changes to models, APIs, or shared logic
  • Interactions surface as required decision points — spec cannot be completed with unresolved interactions
  • Interaction alerts show affected code paths and models
  • Configurable depth for dependency traversal — direct references by default, expandable to transitive dependencies

Visual aids

  • AI can generate quick visual mockups inline during the interview to illustrate UI concepts
  • Visuals are rendered as HTML/CSS inline, not full implementations
  • Figma designs can be imported as reference input to the interview

Spec output

  • Acceptance criteria are extracted from the conversation in real time
  • Output spec is human-readable and directly editable
  • AI can suggest revisions to previously extracted criteria based on new information
  • Spec can be committed to a branch in the target repository when ready
Open questions

Interview structure: Should the interview follow a guided framework (happy path → edge cases → interactions) or be more free-form?

Multi-session flow: When resuming, how much should the AI recap?

Spec granularity: Currently at ticket level — may need to scale to epics or whole projects.
