# WH-008: User identity

**Status:** Not started
**Priority:** Medium
**Team:** Platform

## Summary

Users set their name so changes can be attributed to them. No authentication or permissions system initially.

## Acceptance criteria

- [ ] First-time users are prompted to set their display name
- [ ] Name is stored in the database and associated with all their actions
- [ ] Name persists across sessions (stored in browser + DB)
- [ ] Users can change their name in settings
- [ ] All spec edits, interview messages, and feature changes are attributed to the user
- [ ] No login, passwords, or auth required — trust-based system for now
