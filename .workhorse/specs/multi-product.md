Workhorse manages specs across multiple projects. Projects are the user-facing concept (Tamanu, Tupaia) — each project maps to a GitHub repository under the hood.

## Multi-project

- [ ] Projects can be added by giving them a name and linking a GitHub repository URL
- [ ] Each team belongs to a single project
- [ ] A card's project is determined by its team
- [ ] The AI interviewer loads context from the team's project codebase
- [ ] Specs are committed to the team's project codebase
- [ ] One card always targets exactly one project
- [ ] Project management is in settings
- [ ] The user is always in the context of one project, selected via the project switcher in the sidebar
- [ ] A user can only see projects linked to GitHub repos they have write access to (see WH-021)
